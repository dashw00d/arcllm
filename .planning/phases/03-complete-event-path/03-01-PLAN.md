<objective>
Convert the last two blocking stall sites to event-based chains: Phase 3 merge copies and src1 cache hit path.

Purpose: Phase 3 merge currently calls dev2dev_memcpy_staged_sync which does TWO blocking waits per column per device (q_src.memcpy().wait() + q_dst.memcpy().wait()). The src1 cache hit path silently skips the copy but provides no event for downstream depends_on chains, creating a sync gap. Both must be converted to event-based operation before throughput can reach the 10+ t/s target.

Output: Modified ggml-sycl.cpp and common.hpp with event-based merge and src1 cache event propagation, gated behind GGML_SYCL_ROW_EVENTS=1. Legacy path unchanged.

```cpp
// Fully blocking dev2dev copy using a pre-allocated pinned host staging buffer.
static void dev2dev_memcpy_staged_sync(
        sycl::queue &q_dst,
        sycl::queue &q_src,
        void *ptr_dst,
        const void *ptr_src,
        size_t size,
        void *host_buf) {
    if (host_buf == nullptr) {
        dev2dev_memcpy(q_dst, q_src, ptr_dst, ptr_src, size, "staged_sync_fallback");
        return;

    q_dst.memcpy(ptr_dst, host_buf, size).wait();
}
```

The merge copies GPU_i VRAM -> pinned staging -> dst (host or GPU0 VRAM). The copy direction is OPPOSITE to Phase 1 (Phase 1: GPU0->staging->GPU_i; Phase 3: GPU_i->staging->dst).

```cpp

for (int i = 0; i < ggml_sycl_info().device_count; ++i) {
    if (dev[i].row_low == dev[i].row_high) {
        continue;
    }

    const bool dst_on_device = (i == ctx.device);
    const int64_t row_diff = dev[i].row_high - dev[i].row_low;

    if (dst_on_device) {
        continue;  // main device result is already in place
    }

    ggml_sycl_set_device(i);

    if (use_event_sync) {
        // SYNC-01: Event-based merge — zero host waits.
        // Copy pattern: GPU_i VRAM -> pinned staging -> dst.
        // Chain on dev_events[i] (kernel completion from Phase 1).
        // Reuse staging buffer across columns via e_prev_staging_free.
        queue_ptr src_ooo = ctx.ooo_stream(i);  // source device OOO queue

        // Determine destination queue:
        // - dst_is_host: src_ooo does device->host directly
        // - dst on GPU0: use ooo_stream(ctx.device) for host->GPU0
        const bool dst_is_host = ggml_backend_buffer_is_host(dst->buffer);
        queue_ptr dst_ooo = dst_is_host
            ? src_ooo  // host is accessible from any queue
            : ctx.ooo_stream(ctx.device);

        sycl::event e_prev_merge_staging;
        bool has_prev_merge_staging = false;

        for (int64_t i0 = 0; i0 < ne13*ne12; ++i0) {
            const int64_t i03 = i0 / ne12;
            const int64_t i02 = i0 % ne12;

            float * dst_dd_i = dev[i].dst_dd + (i0*ne1 + src1_col_0) * row_diff;

            void * dst_off_device = dst->data;
            float * dhf_dst_i = (float *) ((char *) dst_off_device + i02*nb2 + i03*nb3);
            GGML_ASSERT(dst->nb[1] == ne0*sizeof(float));
            dhf_dst_i += src1_col_0*ne0 + dev[i].row_low;

            void * staging = dev[i].host_staging_buf;

            for (int64_t col = 0; col < src1_ncols; ++col) {
                float * dst_col = dhf_dst_i + col * ne0;
                float * src_col = dst_dd_i + col * row_diff;
                const size_t merge_bytes = row_diff * sizeof(float);

                // GPU_i -> pinned staging (on source device queue)
                sycl::event e_g2h = src_ooo->submit([&](sycl::handler& h) {
                    // First column of first i0: chain on kernel completion
                    if (!has_prev_merge_staging) {
                        h.depends_on(dev_events[i]);

                        h.depends_on(e_prev_merge_staging);

                        h.memcpy(dst_col, staging, merge_bytes);

                    // Staging -> GPU0 VRAM: cross-device, use dst OOO queue
                    sycl::event e_h2g = dst_ooo->submit([&](sycl::handler& h) {
                        h.depends_on(e_g2h);
                        h.memcpy(dst_col, staging, merge_bytes);
                    });
                    e_prev_merge_staging = e_h2g;
                }
                has_prev_merge_staging = true;
            }

    } else {
        // Legacy blocking merge (unchanged)
        queue_ptr merge_stream = ctx.stream(i, is);

        for (int64_t i0 = 0; i0 < ne13*ne12; ++i0) {
            const int64_t i03 = i0 / ne12;
            const int64_t i02 = i0 % ne12;

            float * dst_dd_i = dev[i].dst_dd + (i0*ne1 + src1_col_0) * row_diff;

            void * dst_off_device = dst->data;
            float * dhf_dst_i = (float *) ((char *) dst_off_device + i02*nb2 + i03*nb3);
            GGML_ASSERT(dst->nb[1] == ne0*sizeof(float));
            dhf_dst_i += src1_col_0*ne0 + dev[i].row_low;

            const bool dst_is_host = ggml_backend_buffer_is_host(dst->buffer);

            if (dst_is_host) {
                for (int64_t col = 0; col < src1_ncols; ++col) {
                    float * dst_col = dhf_dst_i + col * ne0;
                    float * src_col = dst_dd_i + col * row_diff;
                    SYCL_CHECK(CHECK_TRY_ERROR(

                                  row_diff * sizeof(float),
```

**Key design decisions:**

1. **Copy direction:** GPU_i VRAM -> pinned staging -> dst. The source queue (src_ooo) submits the GPU->host copy. The destination queue (dst_ooo) submits the host->GPU copy. For dst_is_host, both copies use src_ooo since host memory is accessible from any queue.

2. **First column chains on `dev_events[i]`** (the Phase 1 kernel completion barrier). This replaces the Phase 2 `dev_events[i].wait()` as the dependency source — the merge copy starts only after the kernel finishes, without a host wait.

3. **Staging buffer reuse** via `e_prev_merge_staging` / `has_prev_merge_staging`. Same pattern as Phase 1's `e_prev_staging_free`. Each GPU->host copy depends_on the prior host->dst completion to prevent staging buffer corruption.

4. **No host waits** inside Phase 3 on the event path. The OOO queue flush at lines 4321-4329 (after Phase 3) is the synchronization point — it waits for ALL submitted merge copies to complete before the next matmul.

5. **Legacy path preserved exactly** in the `else` branch — identical to current code at lines 4258-4312.

**IMPORTANT: The Phase 2 wait loop (lines 4244-4256) stays AS-IS.** Phase 2 dev_events[i].wait() is still needed even with event-based merge because:
- The OOO flush at the end releases event handles — if merge copies are still in flight when flush runs, we get use-after-free
- Phase 2 ensures all kernel work is complete before merge begins reading kernel output
- The flush `ooo_stream(i)->wait()` at lines 4321-4329 then ensures merge copies complete

- Remove or modify the OOO queue flush (lines 4321-4329)
