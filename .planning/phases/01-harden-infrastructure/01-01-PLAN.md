**INFRA-01 — Cmdlist assertion in ggml-sycl.cpp:**

```cpp
if (g_ggml_sycl_row_events) {
    const char * cmdlist_val = getenv("SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS");
    if (cmdlist_val == nullptr || cmdlist_val[0] != '1') {
        GGML_LOG_ERROR(
            "%s: GGML_SYCL_ROW_EVENTS=1 requires "
            "SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=1.\n"
            "  Without immediate command lists, cross-queue event dependencies\n"
            "  never fire because command batches are not flushed when another\n"
            "  queue polls for the event. Inference will hang indefinitely.\n"
            "  Fix: export SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=1\n",
            __func__
        );
        abort();
    }
}
```

Place this BEFORE the `GGML_SYCL_DEBUG("[SYCL] call ggml_check_sycl\n");` line (line 298), immediately after all `get_sycl_env()` reads complete. This ensures the assertion fires at static init time, before model loading begins.

Key details:
- Use `getenv()` directly, NOT `get_sycl_env()` — `SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS` is a system L0 env var, not a GGML env var
- Check `cmdlist_val[0] != '1'` — value "0" is set but should still trigger abort
- Use `GGML_LOG_ERROR` + `abort()` — consistent with GGML_ASSERT pattern, do NOT throw sycl::exception
- Include `%s` format with `__func__` — matches existing GGML_LOG patterns in this file

**INFRA-02 — Context mismatch assertion in common.hpp:**

In the `ooo_stream(int device)` accessor, after `ooo_qptrs[device]` is assigned (lazy init branch), add the context equality check:

```cpp
queue_ptr ooo_stream(int device) {
    if (ooo_qptrs[device] == nullptr) {
        ooo_qptrs[device] = &(dpct::get_device(device).out_of_order_queue());

        if (g_ggml_sycl_row_events) {
            const sycl::context ooo_ctx = ooo_qptrs[device]->get_context();
            const sycl::context io_ctx  = stream(device, 0)->get_context();
            if (ooo_ctx != io_ctx) {
                GGML_LOG_ERROR(
                    "%s: device %d: OOO queue context != in-order stream context.\n"
                    "  Cross-device event dependencies (handler::depends_on) will\n"
                    "  silently fail or be ignored when queues come from different\n"
                    "  SYCL contexts. All queues must share a context.\n"
                    "  This is likely a bug in ooo_stream() construction.\n",
                    __func__, device
                );
                abort();
            }
        }
    }
    return ooo_qptrs[device];
}
```

Key details:
- `sycl::context` has `operator==` in SYCL 2020 (oneAPI 2025.3.2 fully supports this)
- The check fires on FIRST call to `ooo_stream(device)`, which is before any matmul is dispatched
- This assertion is expected to ALWAYS PASS with current dpct queue construction — it is a regression guard for future changes
- Uses `GGML_LOG_ERROR` + `abort()` — same pattern as INFRA-01
- Includes device number in the error message for diagnosis
- `g_ggml_sycl_row_events` is declared `extern` and visible from common.hpp (already used in other headers in this codebase)

Verify `g_ggml_sycl_row_events` is accessible from common.hpp. If not already declared extern, check how other globals like `g_ggml_sycl_debug` are accessed from headers and follow the same pattern.
